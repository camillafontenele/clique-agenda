# Regras de Negócio

Um agendamento deve possuir apenas um status final seguindo esta prioridade:

## 1. Cancelado

Quando:

```ts
cancelledAt != null;
```

Status:

```ts
"cancelled";
```

---

## 2. Agendado

Quando:

```ts
cancelledAt == null && date >= now;
```

Status:

```ts
"scheduled";
```

---

## 3. Finalizado

Quando:

```ts
cancelledAt == null && date < now;
```

Status:

```ts
"finished";
```

---

# Importante

- Um agendamento cancelado nunca deve aparecer como agendado.
- Um agendamento cancelado deve aparecer na seção de finalizados, porém utilizando badge/status visual de cancelado.

# Estrutura da Página

A tela deve conter duas seções:

## 1. Agendados

- Exibir apenas agendamentos futuros não cancelados.
- Ordenar por data crescente.

---

## 2. Finalizados

- Exibir agendamentos passados.
- Exibir agendamentos cancelados.
- Ordenar por data decrescente.

# Tipagem

Criar ou reutilizar tipagem Prisma com include:

- `barbershop`
- `service`

Exemplo:

```ts
Prisma.BookingGetPayload<{
  include: {
    barbershop: true;
    service: true;
  };
}>;
```

# Performance

- Utilize `Promise.all` para consultas paralelas.
- Utilize `include` apenas para dados necessários.
- Ordenar dados diretamente na query do Prisma.
