# Tarefa

Crie um sheet de detalhes/cancelamento de reserva exibido ao clicar em um item de agendamento no componente `@components/booking-item.tsx`.

A interface deve seguir fielmente o Figma:

https://www.figma.com/design/oRcaBF6HX9AxwkIx7txN9q/Aparatus--c%C3%B3pia-?node-id=78-2076&m=dev

# Requisitos Funcionais

- Ao clicar em um `BookingItem`, abrir um Sheet com os detalhes da reserva.
- O Sheet deve exibir os dados da barbearia, serviço, data, horário, preço e telefones.
- Os dados da barbearia e do serviço devem vir do banco de dados.
- A imagem do mapa deve ser estática usando https://www.figma.com/design/oRcaBF6HX9AxwkIx7txN9q/Aparatus--c%C3%B3pia-?node-id=78-2341&t=07gkwwuF9kaDq0TF-4.
- Os botões "Copiar" devem copiar o telefone correspondente para o clipboard.
- Use o componente `@app/barbershops/[id]/_components/copy-button.tsx`.
- O botão "Voltar" deve fechar o Sheet.
- O botão "Cancelar Reserva" deve abrir um Alert Dialog do shadcn confirmando o cancelamento.
- Ao confirmar no Alert Dialog, executar uma server action para cancelar a reserva.
- Cancelar reserva significa preencher o campo `cancelledAt` com a data atual.
- Após cancelar, atualizar a UI para refletir o novo status.

# Regras de Status

Crie uma função utilitária para calcular o status do agendamento.

Prioridade:

1. Se `cancelledAt !== null`
   - status: `"cancelled"`
   - label: `"Cancelado"`

2. Se `cancelledAt === null && date >= now`
   - status: `"scheduled"`
   - label: `"Agendado"`

3. Se `cancelledAt === null && date < now`
   - status: `"finished"`
   - label: `"Finalizado"`

Importante:

- Um agendamento cancelado nunca deve aparecer como agendado.
- Use essa função também em `@components/booking-item.tsx`.
- Exiba badges diferentes para:
  - Agendado
  - Finalizado
  - Cancelado

# Requisitos Técnicos

- Criar uma server action para cancelar a reserva.
- A action deve validar se o usuário está autenticado.
- A action deve validar se a reserva pertence ao usuário logado.
- A action não deve permitir cancelar reserva já cancelada.
- A action não deve permitir cancelar reserva finalizada, se a data já estiver no passado.
- Revalidar a página/rota de agendamentos após o cancelamento.
- Manter tipagem forte com TypeScript.
- Evitar uso de `any`.

# Componentização

- Seguir exatamente espaçamentos, tipografia e layout do Figma.
